Java基础梳理：集合类

发表于 2015-08-10   |   分类于 Java   |   阅读次数

整体结构

Java的集合类可以划分为两个大部分:

  • Iterable，按照一定顺序进行迭代访问的集合类;
  • Map，通过键值对的映射关系进行访问的集合类;

Iterator & ListIterator

  • Iterator 可用来遍历 Set 和 List 集合，但是 ListIterator 只能用来遍历 List 。
  • Iterator 对集合只能是前向遍历，ListIterator 既可以前向也可以后向。
  • ListIterator 实现了 Iterator 接口，并包含其他的功能，比如：增加元素，替换元素，获取前一个和后一个元素的索引，等等。

List & Set & Queue & Map

  • List按照插入的顺序保存元素
  • Set不包含重复的元素
  • Queue按照队列的规则来保存元素，先进先出
  • Map以键值对的形式保存元素，使用键查找指

HashMap & LinkedHashMap & TreeMap

  • HashMap底层将key-value封装成一个Entry对象，用数组来保存Entry。Entry对象的保存位置由hash算法决定。所以HashMap的存储顺序和输入顺序无关。Entry本身是一个链表，若hash位置相同时，Entry.next来存储。使用HashMap时，key值需要正确的重写hashcode和equals方法。
  • LinkedHashMap在基于HashMap的基础上，Entry和HashMap有所不同，是一个包含了before和after的双向链表，同时，LinkedHashMap内部维护一个header的Entry，可以方便的按插入顺序排序或者按访问顺序排序。所以LinkedHashMap可以很方便的实现LRUcache。
  • TreeMap基于红黑树实现，可以按指定排序规则保存元素。使用TreeMap时，key必须实现comparable接口，后者指定Comparator用于排序。

HashSet & LinkedHashSet & TreeSet

  • HashSet基于HashMap实现
  • LinkedHashSet在HashSet的基础上采用LinkedHashMap实现
  • TreeSet基于TreeMap实现

ArrayList & LinkedList & Vector

  • ArrayList基于数组实现，随机访问元素较快，但中间元素的插入和删除操作相对较慢
  • LinkedList基于双向链表实现，插入和删除操作相对更快，随机访问元素较慢
  • Vector和ArrayList一样，基于数组实现，不同的地方在于Vector是线程安全的，采用Synchronized修饰，性能上较ArrayList差；

Vector 属于遗留容器（早期的 JDK 中使用的容器，除此之外 Hashtable、Dictionary、BitSet、Stack、Properties 都是遗留容器），现在已经不推荐使用，但是由于 ArrayList 和 LinkedListed 都是非线程安全的，如果需要多个线程操作同一个容器，那么可以通过工具类 Collections 中的 synchronizedList 方法将其转换成线程安全的容器后再使用（这其实是装潢模式最好的例子，将已有对象传入另一个类的构造器中创建新的对象来增加新功能）。

Fail-fast机制

对于非线程安全的集合类，Java采用Fail-Fast的错误机制来处理并发问题。通过一个modCount变量来记录修改次数，每一次对数据修改都会modCount++，如果在遍历Iterator的过程中modCount发生变化，则抛出ConcurrentModificationException异常。如果在多线程环境下使用集合类，建议采用java.utils.concurrent包下的类。

ConcurrentHashMap

ConcurrentHashMap 是一个并发散列映射表的实现，它允许完全并发的读取，并且支持给定数量的并发更新。相比于 HashTable 和用同步包装器包装的 HashMap（Collections.synchronizedMap(new HashMap())），ConcurrentHashMap 拥有更高的并发性。在 HashTable 和由同步包装器包装的 HashMap 中，使用一个全局的锁来同步不同线程间的并发访问，为阻塞式的，效率较低。ConcurrentHashMap 是设计为非阻塞的。使用Segment分段锁机制，在更新时会局部锁住某部分数据，但不会把整个表都锁住。同步读取操作则是完全非阻塞的。好处是在保证合理的同步前提下，效率很高。坏处是严格来说读取操作不能保证反映最近的更新。
